# 정보통신설비 유지보수·관리 웹앱 기획 정리

## 1. 개요

정보통신설비 유지보수관리제도 시행에 따라, 내부 직원이 유지보수·관리 및 성능점검 대상 건축물을 등록하고, 설비별 점검, 사진 등록, 대가산정, 견적서 및 보고서 출력을 수행할 수 있는 웹사이트/웹앱을 구축한다.

본 시스템은 기존 Cloudflare Pages 기반 웹앱 형태를 고려하며, PC 관리자/검토 화면과 모바일 현장점검 화면을 분리하여 구성한다.

## 3. 관련 정의

### 3.1 정보통신설비

「정보통신공사업법」 제2조제1호에 따른 설비로서 「정보통신공사업법 시행령」 별표 1 ‘공사의 종류’에 따른 설비를 말한다.

### 3.2 관리주체

유지보수·관리 및 성능점검 대상 건축물 등에 설치된 정보통신설비의 소유자 또는 관리자를 말한다.

### 3.3 유지보수·관리

정보통신설비의 기능을 유지하고 이용자의 편의와 안전을 확보하기 위해 정보통신설비를 일상적으로 보수·관리하는 일체의 행위를 말한다.

### 3.4 유지보수·관리자

정보통신설비 유지보수·관리자 인정교육을 받은 기술계 정보통신기술자를 말한다.

### 3.5 성능점검

정보통신설비의 운전·운용 등에 필요한 성능을 점검하는 것을 말한다.

### 3.6 성능점검 대행자

정보통신설비의 성능점검을 대행하는 공사업자 또는 용역업자를 말한다.

## 4. 대상 건축물 및 시설물

### 4.1 대상

- 「건축법」 제2조제2항에 따른 용도별 건축물 중 연면적 5,000㎡ 이상의 건축물
- 「학교시설사업 촉진법」 학교 시설 중 과학기술정보통신부장관이 정하여 고시하는 시설

### 4.2 제외 대상

- 「건축법」 제2조제2항제2호에 따른 공동주택
- 「학교시설사업 촉진법」 제2조제1호에 따른 학교시설

### 4.3 시스템 처리 기준

- 연면적 5,000㎡ 미만 입력 시 “유지보수관리 대상 건축물이 아닙니다.” 알림 표시
- 알림 후 연면적 입력란 초기화
- 건축물 1개당 대가산정 및 견적 1건 생성

## 6. 웹앱 구축 방향

### 6.1 사용 대상

- 내부 직원용 시스템
- 외부 고객사 접속 기능은 우선 제외

### 6.2 로그인 방식

- 아이디 / 비밀번호 방식
- 사용자 권한 구분 필요
- 사용자 생성 및 관리를 위한 메뉴 필요

### 6.3 권한 구조 초안

| 권한 | 주요 역할 |
|---|---|
| 관리자 | 사용자 관리, 기준정보 관리, 전체 데이터 수정 |
| 검토자 | 점검 결과 검토, 보고서 출력, 최종 승인 |
| 점검자 | 모바일 점검 입력, 사진 등록 |

### 6.4 메뉴 구조 초안(해당 메뉴명/메뉴설명을 최우선 적용하여 업무흐름, 기능은 해당 현재 설명을 우선하여 개발에 활용한다)
| 메뉴명 | 메뉴설명 |
|---|---|
| 대시보드 | 등록된 건출물, 건축물 상태별 현황, 기술자 현황, 기술자별 할당된 건축물 개수 |
| 건축물 등록 | 건축물명, 연면적, 주소를 입력하고 연면적 기준에 따른 기술자 등급 및 기술노임단가 산정, 기술자 지정(기술자관리 메뉴에 등록된 기술자), 대상설비별 점검 항목 체크, 설비별 기준인원, 수량입력이 필요한 설비 수량입력, 직접인건비 항목 입력(유류비, 차량운행비, 기타 직접경비), 제경비, 기술료율 선택, 작성한 내용을 토대로 금액 산출하고 등록하는 메뉴|
| 건축물별 점검표 작성 | 건축물 등록을 통해 등록된 건축물을 대상으로 등록 당시 체크된 대상설비별 점검 항목별로 점검 내용 등 점검표를 보여주는데 사용자가 기능점검/성능점검 두개의 점검표를 선택할 수 있게 제공하여야 하며 선택된 점검에 따라 점검표를 보여준다. 기능점검은 연2회, 성능점검은 연1회 실시해야 하므로 한 건축물당 점검표는 3장이 나올수 있으므로 점검 일자별, 기능점검인지 성능점검인지 구분이 되어서 저장되어야 한다. 점검표 구성은 점검위치, 양호/미흡 체크, 점검내용, 사진보드판 형태로 저장이 가능한 사진 촬영/등록 기능을 제공하고 동일한 점검내용으로 점검위치가 여러개일 경우가 있으니 점검위치 추가, 삭제 기능을 포함한 점검 기능을 제공하고 점검이 완료되면 항목 저장 버튼을 통해 점검 유무를 판단, 대상설비별로 점검 유무를 한눈에 볼수 있는 카드 리스트형태의 화면을 점검표 작성시 확인하고 카드를 선택하면 해당 설비의 점검 내용을 작성, 수정 할수 있게 점검표를 보여주는 기능을 제공, 해당 건축물의 점검을 사용자가 완료한 경우 점검표 저장 버튼으로 점검완료 처리하는 기능, 해당 메뉴는 모바일로도 사용이 가능하여야 하므로 PC/모바일 구분하여 메뉴페이지를 작성해야함 |
| 점검표 검수 및 관리 | 점검완료된 건축물별 점검표를 검토하고 검수자가 내용 수정이 가능하고, 검수 완료, 보완 필요 처리를 판단하여 상태 확정(검수를 통해 보완 필요 처리가 된 경우에도 해당 메뉴에서 확인하고 다시 검수하여 검수완료 처리가 가능함) |
| 건축물 관리 | 건축물 등록을 통해 등록된 건축물을 건출묵명 별로 관리(수정/삭제) 하고 건축물별 현재 상태가 등록만 된 상태인지, 점검표를 작성중인 상태인지, 점검이 완료된 상태인지, 보완이 필요한 상태인지, 검수 완료된 상태인지 확인하는 기능으로 세부적으로는 건축물별로 건축물명 클릭시 건축물정보, 점검내역(점검표가 여러개인 경우 점검(기능점검인지 성능점검인지)/점검일자)를 보여줘야 하며, 견적서 출력, 점검표 출력, 보고서 출력 기능을 제공한다. 상태값 판별로는 건축물이 최초 등록된 경우 등록상태, 점검표 작성을 수행하여 하나라도 항목저장이 이루어지는 순간 작성중, 점검표 저장이 이루어진 순간 작성완료, 보완 처리가 된 경우 점검표 보완, 검수 완료가 된 순간 검수 완료 상태를 제공, 예를들어 기능점검 1회 수행하여 검수완료인 상태에서 해당 건축물에 점검표 작성이 새로 생성이 되는경우(기능점검 2회차 또는 성능점검) 점검표 작성중으로 표시가 다시 되어야한다. 이후 상태별 동작은 동일함 |
| AI생성 | 건축물별 대상 설비를 선택하여 점검 내용에 있는 정보를 불러오고 설비별 점검 위치, 수량(수량은 점검위치가 추가된것으로 카운팅)을 참고하여 계통도, 구성도를 GPT API를 이용하여 이미지를 생성 할수 있게 제공, 계통도를 만들것인지 구성도를 만들것인지는 사용자가 선택에 따라 좌우되고 일관성 있는 계통도, 구성도 이미지를 제공이 필요하다.(해당 메뉴는 추후 API키를 입력하여 연동할수 있게만 만들어둔다) |
| 기술자관리 | 상단에 기술자 등급별로 현황 표시하고 이름, 기술자 등급, 연락처, 이메일주소 등 등급별 기술자를 등록/관리 하는 기능 |
| 사용자관리 | ID, 이름, 연락처, 이메일주소, 권한 등 계정 생성 및 관리(수정/삭제) |

## 7. 전체 업무 흐름

```text
로그인
→ 건축물 정보 등록
→ 대상 설비 상세 등록

→ 견적서 엑셀 추출
→ 모바일 현장점검

→ PC 검토
→ 점검표 / 결과보고서 / 성능점검보고서 출력
```

## 9. 설비 상세 등록 기준

- 단순 체크 방식이 아닌 상세 등록 방식 적용
- 설비별 설치 위치, 세부설비, 수량, 관련 파일 등을 관리
- 향후 계통도 또는 구성도를 PDF 또는 이미지로 첨부할 수 있어야 함
- 여러 구역에 나뉘어 설치된 설비라도 대가산정은 1식으로 처리할 수 있음

예시 구조:

```text
건축물
└ 층 / 구역
   └ 설비분류
      └ 설비명
         └ 세부설비
            └ 점검항목
```

## 10. 모바일 점검 기능

### 10.1 기본 방향

- 점검자가 모바일로 현장에서 건축물 및 설비를 선택
- 설비항목별 점검항목과 점검내용 기준에 따라 점검 수행
- 점검결과는 양호 / 미흡으로 판단
- 미흡 시 세부 미흡사항 입력 가능
- 점검항목별 사진 여러 장 등록 가능
- 오프라인 기능은 제외
- GPS 기능은 제외

### 10.2 점검 결과 입력 구조

```text
건축물
└ 설비
   └ 점검항목
      ├ 점검결과: 양호 / 미흡
      ├ 미흡사항
      ├ 점검의견
      └ 사진 여러 장
```

### 10.3 사진 등록 기준

- 점검항목별 여러 장 등록 가능
- 조치 전/후 구분은 적용하지 않음
- 사진에는 보드판 형태의 점검 정보가 자동 표시되어야 함

### 10.4 사진 보드판 표시 정보

## 13.5 직접경비

직접경비는 당해 업무 수행에 필요한 여비, 차량운행비, 현장소요경비 등을 포함한다.

| 항목 | 입력 방식 |
|---|---|
| 여비 | 금액 직접 입력 |
| 차량운행비 | 금액 직접 입력 |
| 현장소요경비 | 금액 직접 입력 |

---

## 13.6 제경비

제경비는 당해 업무의 직접경비에 포함되지 않는 간접비용으로 직접인건비의 110% ~ 120% 한도 내에서 정한다.

- 사용자가 110% ~ 120% 범위 내에서 선택
- 1% 단위 선택 가능

계산식:

```text
제경비 = 직접인건비 × 제경비율
```

---

## 13.7 기술료

기술료는 당해 업무를 위해 점검수행자가 개발·보유한 기술의 사용료 및 이윤 등을 포함한 비용으로, 직접인건비와 제경비를 합한 금액의 20% ~ 40% 한도 내에서 정한다.

- 사용자가 20% ~ 40% 범위 내에서 선택
- 1% 단위 선택 가능

계산식:

```text
기술료 = (직접인건비 + 제경비) × 기술료율
```

---

## 13.8 부가가치세

```text
부가가치세 = (직접인건비 + 직접경비 + 제경비 + 기술료) × 10%
```

---

## 13.9 총 대가

```text
총 대가 = 직접인건비 + 직접경비 + 제경비 + 기술료 + 부가가치세
```

---

## 13.10 직접인건비 계산 방식

### 조정계수 적용 항목

조정계수 적용 여부가 `√`인 항목은 아래 방식으로 계산한다.

```text
직접인건비 = 설비수량 × 기준인원 × 연면적 조정계수 × 기술자 노임단가
```

- `식` 단위 설비는 여러 구역으로 나뉘어 있어도 대가산정은 1식으로 본다.

### 조정계수 미적용 항목

조정계수 적용 여부가 `-`인 항목은 개소 또는 세대 단위로 사용자가 직접 수량을 입력한다.

```text
직접인건비 = 사용자 입력 수량 × 기준인원 × 기술자 노임단가
```

- 사용자 수량 입력은 필수
- 기준인원은 1개소 또는 1세대 기준
